Git? Github? 🤔

  • Git : Linus Torvalds가 개발한 분산 버전 관리 시스템

Git Bash?

  • 셸(Shell)
    • 키보드로 입력한 명령어(Command)를 운영체제(OS)에 전달하여, 그 명령어를 실행하게끔 하는 프로그램
    • e.g. CMD(명령 프롬프트)
  • Git Bash(Bourne Again SHell) :
    • Bourne Shell을 대체하는 자유 소프트웨어
    • Windows 운영체제(OS) 환경에서 리눅스 커맨드를 사용할 수 있게 해 주는 Shell

저장소 clone

  • 내 컴퓨터로 해당 저장소의 모든 파일을 복제함
    • cd Desktop/본인이_만든_폴더명
git clone https://github.com/유저이름/레포이름
  • 해당 경로에 복제되었음을 확인할 수 있음

    작업 후 github에 반영하기

  • 내 컴퓨터에서 파일 수정
  • 내 컴퓨터에서 변경한 사항을 Github 저장소에 반영시켜 줘야 함
    ➔ 어떻게?

status, add, commit, push

cd 방금_clone한_폴더
git status
git add -A
git commit -m "commit 메시지"
git push
  • git status : 변동 사항 확인
  • git add -A : 변동된 사항 전체 추가
  • git commit : 추가한 사항 로컬 반영
  • git push : 로컬에 반영한 사항 Github 저장소에 반영

화재 시 가장 중요한 것 : commit - push

Github 원격 저장소에 반영 완료

pull

  • git pull : 원격 저장소 변경 사항을 로컬에 반영
  • 웹으로 Drag & Drop해서 원격 저장소와 로컬 저장소에 차이가 있는 상황이면, pull 명령어로 변경 사항을 로컬로 가져올 필요가 있음
  • pull 하지 않으면 push 도중 오류 발생 가능

Github의 다른 기능들

  • 버전 관리 : Branch
  • 공동 작업 : Fork, Pull Request(PR)
  • 위키 구축 : Github Wiki
  • 웹 호스팅 : Github pages
  • CI/CD 구축 : Github Actions

Branch

main이 기본 브랜치이나
버전별로 분기하여 별도 관리 가능

Fork

타인의 저장소를 복사해 내 저장소로 가져오는 기능
➔ Pull Request를 위한 준비 작업

Pull Request(PR)

Fork해 온 awesome-sushi 저장소 일부를 수정해 PR
➔ 지나가던 사람이 무료로 기여해 주는 생태계

Github Pages

  • 본인이름.github.io로 저장소를 만들면 자동으로 해당 주소를 통한 웹 호스팅 기능 제공
  • 저장소를 만든 뒤 간단한 HTML 파일을 넣어 확인해 보자!

html 예시

  1. 코드 메모장에 복사, 붙여넣기 후 html 확장자로 저장
  2. 간단하게 Drag & Drop으로 본인이름.github.io 저장소에 올리기
  3. 주소창에 본인이름.github.io를 쳐서 접속 (시간 소요)

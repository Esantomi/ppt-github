Git Bash?

  • 셸(Shell)
    • 키보드로 입력한 명령어(Command)를 운영체제(OS)에 전달하여, 그 명령어를 실행하게끔 하는 프로그램
    • e.g. CMD(명령 프롬프트)
  • Git Bash(Bourne Again SHell) :
    • Bourne Shell을 대체하는 자유 소프트웨어
    • Windows 운영체제(OS) 환경에서 리눅스 커맨드를 사용할 수 있게 해 주는 Shell

    • git config --global user.name "내 이름"
git config --global user.email "가입한_이메일@example.com"

저장소 clone

  • 내 컴퓨터로 해당 저장소의 모든 파일을 복제함
    • cd Desktop/본인이_만든_폴더명
git clone https://github.com/유저이름/레포이름
  • 해당 경로에 복제되었음을 확인할 수 있음

    작업 후 github에 반영하기

  • 내 컴퓨터에서 파일 수정
  • 내 컴퓨터에서 변경한 사항을 Github 저장소에 반영시켜 줘야 함
    ➔ 어떻게?

status, add, commit, push

cd 방금_clone한_폴더
git status
git add -A
git commit -m "commit 메시지"
git push
  • git status : 변동 사항 확인
  • git add -A : 변동된 사항 전체 추가
  • git commit : 추가한 사항 로컬 반영
  • git push : 로컬에 반영한 사항 Github 저장소에 반영

화재 시 가장 중요한 것 : commit - push

Github 원격 저장소에 반영 완료

pull

  • git pull : 원격 저장소 변경 사항을 로컬에 반영
  • 웹으로 Drag & Drop해서 원격 저장소와 로컬 저장소에 차이가 있는 상황이면, pull 명령어로 변경 사항을 로컬로 가져올 필요가 있음
  • pull 하지 않으면 push 도중 오류 발생 가능
